# ระบบจัดการหอพัก 30 ห้อง

ระบบจัดการหอพัก 3 ชั้น ชั้นละ 10 ห้อง พร้อมบันทึกรายรับ-รายจ่ายและรายงานการเงิน
ทำงานบนเครื่องตัวเองทั้งหมด **ไม่ต้องติดตั้งอะไรเพิ่ม** นอกจาก Python

---

## เริ่มใช้งาน

**ดับเบิลคลิกที่ไฟล์ `เปิดระบบ.bat`** — ระบบจะเปิดเบราว์เซอร์ให้อัตโนมัติที่
<http://localhost:8765>

ครั้งแรกที่เปิด ระบบจะสร้างห้อง 101–110, 201–210, 301–310 ให้เอง 30 ห้อง

**หน้าต่างสีดำต้องเปิดค้างไว้ตลอดเวลาที่ใช้งาน** ถ้าปิดหน้าต่างนั้น ระบบจะหยุดทำงาน
(ข้อมูลไม่หาย — บันทึกลงไฟล์ทันทีทุกครั้งที่กดบันทึก)

หากกดเปิดระบบซ้ำโดยยังไม่ได้ปิดตัวเก่า ไฟล์ `.bat` จะ**ปิดตัวเก่าให้อัตโนมัติ**ก่อนเปิดใหม่
จึงมั่นใจได้ว่าที่เห็นบนหน้าจอคือเวอร์ชันล่าสุดเสมอ

---

## เปิดจากมือถือ

มี 2 โหมด เลือกตามว่าอยู่ที่ไหน

### อยู่ในหอ — `เปิดระบบ.bat /lan`

เข้าจากมือถือที่ `http://<IP ของคอมพิวเตอร์>:8765` ไม่ต้องติดตั้งอะไรเพิ่ม

> ⚠️ โหมดนี้เปิดให้ **ทุกเครื่องในวง Wi-Fi นั้น** เข้าได้ ใช้ได้เฉพาะ Wi-Fi ของหอ
> เท่านั้น **ห้ามใช้กับ Wi-Fi ร้านกาแฟหรือที่สาธารณะ** เพราะระบบไม่มีระบบล็อกอิน

### อยู่ที่ไหนก็ได้ — `เปิดระบบ.bat /phone`

ใช้ผ่าน **Tailscale** ซึ่งเป็นเครือข่ายส่วนตัวที่มีเฉพาะเครื่องของเราเอง
ระบบจะเปิดรับเฉพาะช่องทางนี้ช่องทางเดียว เครื่องอื่นในวง Wi-Fi เดียวกันจะมองไม่เห็นเลย

ตั้งค่าครั้งเดียว:

1. ที่คอม — ติดตั้ง Tailscale แล้วล็อกอิน (ไอคอนมุมขวาล่างต้องขึ้น Connected)
2. ที่มือถือทุกเครื่องที่จะใช้ — โหลดแอป Tailscale จาก App Store / Play Store
   แล้ว **ล็อกอินด้วยบัญชีเดียวกัน**
3. ที่คอม — เปิดด้วย `เปิดระบบ.bat /phone` หน้าจอสีดำจะบอกที่อยู่ที่ขึ้นต้นด้วย `100.`
4. ที่มือถือ — เปิด Tailscale ค้างไว้ แล้วเข้าเบราว์เซอร์ตามที่อยู่นั้น

หลังจากนั้นใช้ได้จากทุกที่ ทั้ง 4G/5G และ Wi-Fi ที่อื่น โดยข้อมูลไม่ได้ออกสู่
อินเทอร์เน็ตสาธารณะ (ดู `docs/adr/0007`)

> **มือถือหาย** ให้เข้า [login.tailscale.com](https://login.tailscale.com) แล้วลบ
> เครื่องนั้นออก เครื่องที่ถูกลบจะเข้าระบบไม่ได้อีกทันที

### ข้อมูลอัพเดทเอง

ทุกหน้าจะเช็คทุก 3 วินาทีว่ามีข้อมูลใหม่ไหม ถ้ามีจะโหลดใหม่ให้เอง —
บันทึกรับชำระที่คอม มือถือที่เปิดค้างไว้จะเห็นทันทีโดยไม่ต้องกดรีเฟรช

ยกเว้นหน้าที่**กำลังกรอกข้อมูลค้างอยู่** (เช่นหน้าจดมิเตอร์) ระบบจะไม่โหลดทับ
แต่จะขึ้นปุ่มให้กดเองเมื่อพร้อม — ที่กรอกไว้จะไม่หาย

## ขั้นตอนใช้งานประจำเดือน

| ลำดับ | ทำอะไร | ไปที่หน้า |
| --- | --- | --- |
| 1 | เดินจดมิเตอร์น้ำ-ไฟทุกห้อง กรอกลงตารางหน้าเดียวจบ | **จดมิเตอร์** |
| 2 | กดปุ่มเดียวออกใบแจ้งหนี้ให้ทุกห้องที่มีผู้เช่า | **ใบแจ้งหนี้** |
| 3 | ส่งบิลให้ผู้เช่า (กด "พิมพ์" ที่ใบแจ้งหนี้ หรือแคปหน้าจอส่ง LINE) | **ใบแจ้งหนี้** → คลิกเลขที่บิล |
| 4 | ผู้เช่าโอนเงินมา → กด "รับชำระ" | **ใบแจ้งหนี้** |
| 5 | มีค่าซ่อม ค่าน้ำรวมของตึก ค่าจ้าง → บันทึกไว้ | **รายจ่าย** |
| 6 | ดูว่าเดือนนี้ได้เท่าไร เหลือเท่าไร ใครค้าง | **ภาพรวม** / **รายงานการเงิน** |

**หน้าภาพรวม** จะบอกทุกครั้งว่ายังเหลืออะไรต้องทำ เช่น "ยังไม่ได้จดมิเตอร์ 4 ห้อง"
หรือ "จดแล้วแต่ยังไม่ออกบิล 2 ห้อง"

## สิ่งที่ระบบทำให้

- **ผังห้อง 30 ห้อง** แยกตามชั้น เห็นทันทีจากสีทั้งช่อง
  **แดงเข้ม = ห้องว่าง** (ยังไม่มีรายได้) · **เขียว = มีผู้เช่าอยู่** ·
  **แถบเหลืองใต้ช่อง = ค้างชำระ** พร้อมยอดเงินและจำนวนวันที่เกินกำหนด
- **คิดค่าน้ำค่าไฟอัตโนมัติ** จากเลขมิเตอร์ × อัตราต่อหน่วย
- **ใบแจ้งหนี้/ใบเสร็จพิมพ์ได้** มีชื่อหอพัก ที่อยู่ พร้อมเพย์
- **รับชำระบางส่วนได้** จ่ายมา 2,000 จาก 4,880 ระบบจำได้ว่าค้างเท่าไร
- **ค่าปรับล่าช้า** คิดตามจำนวนวันที่เกิน หลังหักวันผ่อนผัน (กดเองเมื่อต้องการ)
- **รายงานการเงิน** รายรับ-รายจ่าย-กำไรสุทธิ รายเดือนและรายปี อัตราการเช่า
  อัตราเก็บเงินได้ และผลตอบแทนรายห้อง
- **อายุหนี้ค้าง** แยก 1-30 / 31-60 / 61-90 / เกิน 90 วัน — รู้ว่าต้องตามใครก่อน
- **ส่งออก Excel** ทั้งปี 5 ชีต ตัวเลขเป็นตัวเลขจริง เอาไปทำต่อได้

## เรื่องเงิน: ตัวเลขที่ต้องเข้าใจ

ระบบแยก **"ออกบิล"** กับ **"เก็บได้"** ออกจากกันเสมอ

- **ออกบิล (invoiced)** = ยอดที่แจ้งหนี้ไปในเดือนนั้น
- **เก็บได้ (collected)** = เงินที่เข้ามาจริงในเดือนนั้น ตรงกับ statement ธนาคาร
- **กำไรสุทธิ** = เก็บได้ − รายจ่าย

บิลที่ออกเดือนมิถุนายนแต่จ่ายเดือนกรกฎาคม จะนับเป็นรายรับของ**เดือนกรกฎาคม**
ส่วนต่างระหว่างสองตัวเลขนี้คือ**ยอดค้างชำระ** ซึ่งเป็นตัวเลขที่บอกสุขภาพของกิจการจริง ๆ

## ตั้งค่าอัตราต่าง ๆ

ไปที่หน้า **ตั้งค่า** เพื่อแก้ชื่อหอพัก ที่อยู่ พร้อมเพย์ ค่าน้ำ/หน่วย ค่าไฟ/หน่วย
ค่าส่วนกลาง วันครบกำหนดชำระ และค่าปรับล่าช้า

> อัตราที่แก้มีผลกับ**บิลที่ออกใหม่เท่านั้น** ใบที่ออกไปแล้วเก็บอัตราของตัวเองไว้
> จึงไม่ถูกแก้ย้อนหลัง (ดู `docs/adr/0002`)

ถ้าผู้เช่ารายไหนตกลงอัตราพิเศษไว้ ให้กำหนดที่สัญญาเช่าของรายนั้นแทน

## สำรองข้อมูล

ข้อมูลทั้งหมดอยู่ในไฟล์เดียว: **`data/apartment.db`**

คัดลอกไฟล์นี้ไปเก็บที่อื่น = สำรองข้อมูลครบถ้วน หรือกดปุ่ม **"สำรองข้อมูลตอนนี้"**
ในหน้าตั้งค่า ระบบจะสร้างสำเนาไว้ใน `data/backups/` พร้อมวันเวลา

**ควรสำรองทุกเดือนหลังออกบิลเสร็จ** และคัดลอกออกไปไว้นอกเครื่องด้วย

## ใช้ผ่านบรรทัดคำสั่ง (ถ้าถนัด)

```powershell
python -m apartment serve                  # เปิดหน้าเว็บ
python -m apartment init                   # สร้างฐานข้อมูลและห้อง
python -m apartment invoice 2026-07        # ออกบิลทั้งงวด
python -m apartment report --period 2026-07  # สรุปขึ้นหน้าจอ
python -m apartment export --year 2026     # ออกไฟล์ Excel
python -m apartment backup                 # สำรองฐานข้อมูล
```

ฐานข้อมูลเป็น SQLite มาตรฐาน เปิดด้วยเครื่องมือ SQL อะไรก็ได้เพื่อ query เพิ่มเติม
ตัวเลขจะตรงกับหน้าเว็บเสมอ (จำไว้ว่าคอลัมน์เงินเก็บเป็น**สตางค์** หารด้วย 100 เป็นบาท)
